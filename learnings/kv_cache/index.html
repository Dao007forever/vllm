vLLM KV Cache — Learning Notes

Notes on how vLLM allocates, hashes, and reuses KV memory. Generated from the markdowns in learnings/kv_cache/.

What's here

Seven docs covering: the prefix-caching design, the block/group/column-packing layout, fifteen invariants you can't break without silent corruption, the kernel-tile alignment that forces inflated block_size, and how Mamba's recurrence reuses the same allocator with completely different semantics.

New here? Start with the Beginner's Guide. It builds the intuition for everything else.

Top-level (this folder)

  • BEGINNER_GUIDE.html 15-section on-ramp: blocks, prefix cache, groups, column packing, free list, ref counting, preemption, sentinels, linear attention, end-to-end walk-through.
  • KV_CACHE.html Prefix caching design (local + connector). The receptive-field wall: why "suffix-digest" hashing is silently unsound for any transformer of depth ≥ 2.
  • KV_CACHE_ALLOCATION.html Why groups exist, why they need uniform physical block size, how column packing pools physical memory across groups. Includes a hybrid Full+SWA worked example.
  • KV_CACHE_ASSUMPTIONS.html 15 hidden assumptions and what would silently break if you changed them: blocks, block_size, layout, sentinels, post-forward hashing, free queue, lock-free ref counts, NONE_HASH randomization, min-clamp, preemption.
  • KERNEL_BLOCK_ALIGNMENT.html Why CUTLASS MLA on SM100 asserts block_num % (128 / PAGE_SIZE) == 0, and how vLLM inflates block_size to satisfy it (vs the page-table padding alternative).
  • EAGLE.html PR #44082: how a memory optimization for SWA prefix caching silently broke all EAGLE/MTP cache hits, and the fix that caches exactly the one lookahead block past each aligned boundary.
  • MTP_EAGLE_NEED_SHIFT.html PR #43447: chart-first explanation of SWA need and shift, using the 127-token MTP/EAGLE replay-boundary example.
  • LINEAR_ATTENTION.html Why Mamba/RWKV/RetNet break the cache abstraction's positional retrieval. Eight properties trace what changes and what stays the same.
  • MAMBA.html What's awkward about Mamba specifically: two state tensors, selectivity, two kernels, conv-state alignment, spec-decode rollback, no DCP/PCP.

Blue tier covers the original-design (softmax-attention) lineage. Green tier is a kernel-side correction. Orange tier generalizes the abstraction to recurrent state. Skipping the violet tier and diving into orange directly is the most common reason readers get confused — the terminology and the abstraction boundary are different at every layer.
Mental model
  • BEGINNER_GUIDE teaches the vocabulary — blocks, groups, slots, column tensors, ref counts, null blocks.
  • KV_CACHE is the design doc: the prefix-cache flow and the correctness argument for why today's hash is the only sound one.
  • KV_CACHE_ALLOCATION is the layout doc: from "why have groups" to "why column packing is the unique solution."
  • KV_CACHE_ASSUMPTIONS is the contracts doc: 15 invariants whose violation produces silent corruption.
  • KERNEL_BLOCK_ALIGNMENT zooms into one specific kernel-side constraint (CUTLASS MLA) and how vLLM resolves it.
  • LINEAR_ATTENTION reframes the cache problem for recurrent state.
  • MAMBA is the most concrete linear-attention case — selectivity, two kernels, conv-state subtlety.
